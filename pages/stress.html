Data Storage & Access Justification

Every piece of data in this project is stored in the structure that best fits its access pattern.

Data Structure Justification
Users
HashTable (open addressing)		 Fixed set of 10 users inserted once, never deleted. Keyed by username string. O(1) average lookup by username, the most frequent operation. Linear probing keeps cache locality and the load factor stays low (10/32 ≈ 31%), so collisions are rare.
Active Listings
std::map<time_t, vector<Listing*>>		 tree keyed by expiry timestamp; each key maps to a bucket of listings that expire at the same second. Listings must be checked for expiry in time order. A sorted map makes the earliest-expiring listing always the first entry, so the server can close auctions by iterating from the front without scanning the entire collection.
Bids per Listing
std::vector<Bid*> (append-only)		 Each Listing owns a dynamic array of bid pointers, one entry per bid in chronological order. Bids are only ever added (never removed or reordered). Append is O(1). The highest bid is always the last element, so reading it is O(1) with no search needed.
Sold Listings
std::vector<Listing*>		 A flat array of pointers to listings that have closed. Sold listings are only appended and read sequentially (e.g., for the sold page). A vector is the simplest structure that fits this append-and-scan pattern with minimal overhead.
Event Log
std::vector<string>		 A shared, append-only list of timestamped log strings, protected by a mutex. Log entries are always appended and displayed in insertion order. A vector gives O(1) append and O(n) sequential read with no wasted complexity. The mutex ensures thread safety without performance overhead on a small log.
Persistent State
data/listings.json (flat file)		 All active listings are serialized to a single JSON file on disk via saveToFile(). The dataset is small and writes are infrequent (triggered by bids and buyouts), so a flat JSON file is sufficient, no database overhead needed. On startup the server reads this file to restore state, giving simple crash recovery.
